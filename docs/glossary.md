# Glossary

Terms specific to FridgeCheck (or used in non-obvious ways here). When in doubt, the code is the source of truth.

---

### Access token

Short-lived JWT (HS256) signed with `SECRET_KEY`. Default 30-minute lifetime (`ACCESS_TOKEN_EXPIRE_MINUTES` in `app/config.py`). Carries `{sub, exp, iat, type:"access"}`. Not stored server-side. Sent on every request as `Authorization: Bearer <token>`.

### Active item

A row in `pantry_items` with `deleted_at IS NULL`. Soft-deleted rows are excluded from every standard query. Two partial indexes (`idx_pantry_items_household_active`, `idx_pantry_items_expiry_active`) make the active subset cheap to query.

### Added_by

The user who logged a pantry item. FK to `users.id` with `ON DELETE SET NULL` — if a user is removed, their items survive but lose attribution. Different from "household owner" (which is per `household_members.role`).

### APScheduler

In-process Python cron-like scheduler. We run it inside the dedicated `worker` container, not the API container, so a long-running job can't tie up an API task. Two jobs today: daily `expiry_check` at 08:00 UTC and weekly `cleanup` at Sunday 03:00 UTC.

### Bearer token

The HTTP `Authorization: Bearer <access_token>` scheme. We use it for all protected endpoints. Refresh tokens are **not** sent in this header — they go in request bodies to `/auth/refresh` and `/auth/logout`.

### Category (item)

One of 12 enum values: `produce`, `dairy`, `meat`, `seafood`, `bakery`, `frozen`, `canned`, `dry_goods`, `beverages`, `condiments`, `snacks`, `other`. Defined as `item_category_enum` in PostgreSQL and mirrored in `ItemCategory` (Python) + `ItemCategory` (TypeScript). Adding a value requires both an Alembic migration with `ALTER TYPE ... ADD VALUE ...` and TS/Python updates.

### Conventional Commits

Commit-message format (`<type>(<scope>): <description>`) we use because `release.yml` parses it to bump semver. See [`development-workflow.md#commits--conventional-commits`](development-workflow.md#commits--conventional-commits).

### Correlation ID

UUID attached to every request as `X-Correlation-ID`. Generated by the middleware in `main.py` if the client doesn't send one. Bound to structlog's contextvars so every log line in the request lifecycle carries it. Use it to trace a request across api + worker logs.

### Design tokens

CSS custom properties defined in `frontend/src/index.css` `:root`. Mapped to Tailwind utility classes via `tailwind.config.js`. Components reference them through Tailwind, never raw `var()` or hex. The single source of truth for visual design is `design-system/fridgecheck/MASTER.md`.

### Expiring soon

Items with `expiry_date <= today + N days` and `deleted_at IS NULL`. The default `N` is 3 days for `GET /items/expiring` and the rescue-recipes precheck. The expiry worker checks against `settings.notification_thresholds` (`[1, 3]` by default).

### Fresh

Urgency tier where the item has more than 7 days until expiry. See [Urgency](#urgency).

### Household

The unit of pantry ownership. Many users belong to a household via `household_members`; many items belong to a household via `pantry_items.household_id`. Every authenticated operation that touches items goes through `require_household_member`.

### Household member

A row in `household_members` linking a user to a household with a role (`owner` or `member`). `(household_id, user_id)` is uniquely constrained. Owners are the only role allowed to regenerate the invite token; everything else is symmetric.

### HouseholdError / AuthError / domain error

Service-layer exceptions raised by `services/<x>_service.py`. Routes catch them and translate to `HTTPException`. Don't raise them from routes; don't raise `HTTPException` from services (the rescue-recipe service is the one intentional exception).

### Idempotent item add

`POST /households/{id}/items` doesn't create a duplicate when an item with the same `name` already exists in the household. It increments the existing item's `quantity` instead. See `ItemService.add_item` (`backend/app/services/item_service.py`).

### Invite token

64-character URL-safe random string (`secrets.token_urlsafe(32)`) stored on the `households` row. Acts as a bearer credential for joining the household. **No expiry by default**; owners regenerate to invalidate. The associated invite *link* is `<base_url>/join?token=<invite_token>`. The frontend's join page is `/join/:token`.

### JWT

JSON Web Token. We use HS256-signed JWTs for access tokens only. Refresh tokens are opaque UUIDs, not JWTs — they're stored server-side in `refresh_tokens` so they can be revoked.

### Member role

`owner` or `member`. Stored on `household_members.role`. The user who creates a household is `owner`; everyone who joins via invite is `member`. Today only one operation is owner-restricted (regenerate invite). **TODO(verify)** whether ownership transfer / multi-owner is on the roadmap.

### nginx

Reverse proxy in `docker-compose.yml`. In production it proxies `/api/*` to the api task and serves the SPA's static build. In dev it nominally serves the same role, but the dev `frontend` container runs Vite on `:5173` separately — see the TODO(verify) in `onboarding.md` about dev nginx wiring.

### Open Food Facts (OFF)

Free, crowdsourced food product database (<https://world.openfoodfacts.org>). The frontend looks up barcodes here directly (`src/api/openFoodFacts.ts`) to prefill item name and category. There is no backend wrapper.

### Pantry item

A row in `pantry_items`. Synonymous with "item" everywhere in this codebase. Always scoped to a household. Has a `category`, `quantity`, `unit`, `added_date`, `expiry_date`, optional `notes`, soft-deletable.

### PWA

Progressive Web App. Configured via `vite-plugin-pwa` (Workbox) in `frontend/vite.config.ts`. Generates a service worker that caches `js/css/html/ico/png/svg` and applies a NetworkFirst strategy to API responses. Lets users "install" FridgeCheck to their home screen.

### Rate limit

Sliding-window via Redis, keyed by client IP. Default 100 requests / 60 seconds (`settings.rate_limit_requests`, `settings.rate_limit_window_seconds`). Implemented as middleware in `app/main.py`. Bypassed for `/health`, `/api/docs`, `/api/redoc`, `/api/openapi.json`. Fails open if Redis is down.

### Recipe suggestion (legacy)

The TheMealDB-backed `GET /households/{id}/recipes`. Returns up to 10 meals filtered by the household's expiring ingredients, scored by overlap. Falls back to a hard-coded list on upstream failure. **Distinct from Rescue Recipes.**

### Refresh token

Opaque UUID4 issued alongside an access token. Lifetime defaults to 7 days (`REFRESH_TOKEN_EXPIRE_DAYS`). Stored server-side in `refresh_tokens` so it can be revoked. Each refresh **rotates** the token — the old one is marked `revoked_at` and a new one is issued.

### Rescue recipes

`POST /api/v1/recipes/rescue` — Anthropic-powered (`claude-sonnet-4-20250514`) recipe suggestions for using up items expiring within 3 days. Requires 3+ such items. Backend-only; the API key never reaches the browser. Cached in Redis for 10 minutes per household + item-set hash.

### Service

Backend layer that holds business logic. One class per resource (`AuthService`, `HouseholdService`, `ItemService`, `RecipeService`). Takes `db: AsyncSession` in `__init__`, calls into repositories, raises domain errors. Routes are thin wrappers over services.

### Repository

Backend layer that holds SQL only. One class per aggregate (`UserRepository`, `HouseholdRepository`, `ItemRepository`). Takes `db: AsyncSession` in `__init__`, returns ORM objects or primitives. Never raises HTTP errors; never contains business logic.

### Soft delete

Setting `deleted_at = now()` instead of `DELETE FROM`. Applies to `pantry_items`. Makes accidental deletions recoverable for 7 days, after which the worker's weekly cleanup job hard-deletes them. Every "active items" query must filter `deleted_at IS NULL`.

### structlog

Our logging library. Use it instead of `print()` or stdlib `logging`. Pattern:

```python
import structlog
logger = structlog.get_logger(__name__)
logger.info("household.created", household_id=str(h.id), user_id=str(user.id))
```

Snake-case dot-namespaced events. Pass structured kwargs, not f-strings. Never log secrets, tokens, or full request bodies.

### TanStack Query

The frontend's server-state library (formerly React Query). One hook file per resource in `src/hooks/`. Query keys are tuples scoped by resource id. `AuthContext.login/register/logout` calls `queryClient.clear()` to drop cached server state from the previous session.

### Today (urgency)

Urgency tier where `expiry_date == today`. Renders with the danger colour token. See [Urgency](#urgency).

### This_week (urgency)

Urgency tier where the item expires in 1–7 days. Renders with the warning colour token. See [Urgency](#urgency).

### Urgency

Computed (not stored) field on `ItemResponse` derived from `expiry_date` vs. today:

| Urgency | Days until expiry |
|---------|-------------------|
| `expired` | `< 0` |
| `today` | `== 0` |
| `this_week` | `1–7` |
| `fresh` | `> 7` |

Computed in `ItemService.compute_urgency` (`backend/app/services/item_service.py`). The frontend has its own grouping helper in `src/utils/urgency.ts` that mirrors the same logic.

### ZXing

In-browser barcode scanning library (`@zxing/library`). Lazy-loaded by `BarcodeScanner.tsx` (it's ~600 KB, so don't pull it into the main bundle). Reads barcodes from the device camera; we then look the resulting EAN up against Open Food Facts.
